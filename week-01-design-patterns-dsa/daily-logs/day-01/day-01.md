# Day 01 - SOLID Principles

**Date:** June 10, 2026

## Objective

To understand and implement the five SOLID principles of Object-Oriented Design using Java.

---

## Topics Covered

### 1. Single Responsibility Principle (SRP)

* Learned that a class should have only one reason to change.
* Separated business logic from reporting responsibilities.
* Improved maintainability and readability of code.

### 2. Open/Closed Principle (OCP)

* Understood that software entities should be open for extension but closed for modification.
* Implemented abstraction using interfaces.
* Added support for new payment methods without modifying existing code.

### 3. Liskov Substitution Principle (LSP)

* Learned that derived classes should be substitutable for their base classes.
* Identified inheritance structures that violate expected behavior.
* Refactored examples to ensure proper substitutability.

### 4. Interface Segregation Principle (ISP)

* Understood that clients should not be forced to implement methods they do not use.
* Split large interfaces into smaller, focused interfaces.
* Implemented role-specific interfaces for different animal behaviors.

### 5. Dependency Inversion Principle (DIP)

* Learned that high-level modules should depend on abstractions rather than concrete implementations.
* Used interfaces to decouple business logic from implementation details.
* Improved flexibility and testability of the application.

---

## Practical Implementations

Implemented Java examples demonstrating:

* SRP using separate service and report classes.
* OCP using payment strategy interfaces.
* LSP using correct inheritance hierarchies.
* ISP using segregated behavior interfaces.
* DIP using dependency injection through abstractions.

---

## Challenges Faced

* Initially understanding the difference between OCP and DIP.
* Identifying real-world examples of LSP violations.
* Designing interfaces that follow ISP without over-segmentation.

---

## Key Learnings

* SOLID principles improve code maintainability and scalability.
* Proper abstraction reduces coupling between components.
* Interface-driven design enables easier future enhancements.
* Following SOLID principles makes applications easier to test and maintain.

---

## Outcome

Successfully completed the study and implementation of all five SOLID principles with Java examples and documented the concepts for future reference.
